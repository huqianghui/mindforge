# 控制论 Agent 设计（Cybernetics-Agent Design）

## 摘要

基于金观涛《控制论与科学方法论》的核心框架，提出 AI Agent 与控制系统的**结构同构**命题：Agent = 负反馈控制系统 + 共轭变换 + 随机探索。这不是类比，而是同构——控制论的三个基本机制恰好对应 Agent 设计的三个支柱。

由此推导出 Harness Engineering 的五层设计维度（控制论五层）和十条方法论硬规则，并与六层工程模型、三层控制模型形成互补的三套分析框架（WHY / WHAT / HOW）。

## Claims

### Claim: Agent 与控制系统是结构同构，不是类比

- **来源**：[[控制论与科学方法论——从控制论到AI Agent设计方法论]]
- **首次出现**：2026-04-17
- **最近更新**：2026-04-17
- **置信度**：0.8
- **状态**：active

> 控制论三个基本机制——负反馈（收敛）、共轭变换（表示）、随机探索（逃逸）——恰好对应 Agent 的评估-修正循环、问题表示与拆解、多路径尝试与策略切换。

### Claim: 共轭变换是 Agent 失败的首要原因——表示错误 > 推理错误

- **来源**：[[控制论与科学方法论——从控制论到AI Agent设计方法论]]
- **首次出现**：2026-04-17
- **最近更新**：2026-04-17
- **置信度**：0.8
- **状态**：active

> 如果 Agent 很"聪明"但做不对事，90% 是共轭变换设计错了——不是推理问题，是表示问题。方法论原则：先设计 L（共轭变换），再写 prompt。

### Claim: 控制能力公式——Agent 失败 ≠ 模型不行

- **来源**：[[控制论与科学方法论——从控制论到AI Agent设计方法论]]
- **首次出现**：2026-04-17
- **最近更新**：2026-04-17
- **置信度**：0.8
- **状态**：active

> 控制能力 = 信息 × 工具 × 反馈 × 表示。Agent 失败 90% 是控制能力不够（信息不足、工具不足、反馈太弱、表示不当），不是模型不行。

### Claim: 控制论五层是正交维度，不是串行层级

- **来源**：[[控制论与科学方法论——从控制论到AI Agent设计方法论]]
- **首次出现**：2026-04-17
- **最近更新**：2026-04-17
- **置信度**：0.8
- **状态**：active

> 六层架构和三层模型是模块/层级划分（有边界、有接口），控制论五层更像是每个模块都需要考虑的设计维度。如"共轭变换"不只存在于 L3 Skills，而是 L2 任务分解、L3 能力抽象、L4 工具调用中都有体现。

### Claim: L5 Runtime 是控制论框架的映射缺位——状态观测器

- **来源**：[[控制论与科学方法论——从控制论到AI Agent设计方法论]]
- **首次出现**：2026-04-17
- **最近更新**：2026-04-17
- **置信度**：0.7
- **状态**：active

> 六层模型的 L5 Runtime（状态追踪、上下文管理、资源隔离、检查点快照）在控制论五层中无显式对应。控制论假设"状态是已知的"，但工程中"谁来维护 current_state"恰恰是最重的基础设施。L5 Runtime 是控制系统的状态观测器（State Observer），建议增设 L0 状态观测层或将其作为正交维度。

### Claim: 三套框架互补——搭系统用六层，调系统用三层，设计决策回控制论

- **来源**：[[控制论与科学方法论——从控制论到AI Agent设计方法论]]
- **首次出现**：2026-04-17
- **最近更新**：2026-04-17
- **置信度**：0.8
- **状态**：active

> 控制论五层（WHY，设计原理）、六层工程模型（WHAT，模块结构）、三层控制模型（HOW，运行动力学）三者回答不同问题。收敛性公式 = 约束强度 × 反馈频率 / 执行自由度，其中每个因子的来源由控制论五层解释。
