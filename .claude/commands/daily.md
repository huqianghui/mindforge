创建今天的日记，把前面一天没有完成的事项复制过来。

具体步骤：

1. **首先用 Bash 工具运行 `date +"%Y-%m-%d %A"` 获取真实的当前日期和星期几**（不要依赖系统上下文中的日期，可能不准确）。将星期映射为中文：Monday=周一, Tuesday=周二, Wednesday=周三, Thursday=周四, Friday=周五, Saturday=周六, Sunday=周日。文件名格式为 `YYYY-MM-DD-周X.md`（如 `2026-04-01-周三.md`）
2. 查找前一天的日记文件。**位置规则：当月日记直接放在 `daily-work-item/` 根目录，过往月份归档在 `daily-work-item/YYYYMM/` 子目录**。查找时必须同时覆盖根目录和所有月份子目录，例如：
   ```
   ls daily-work-item/2026-*.md daily-work-item/2026*/2026-*.md 2>/dev/null | sed 's|.*/||' | sort | tail -3
   ```
   ⚠️ 不要用 `ls daily-work-item/ | sort | tail`——字典序会把 `2026-07-xx.md` 排在 `202603/` 目录名之前，导致漏掉根目录的当月日记
3. 读取前一天日记内容，从「追踪任务」和「今天主任务」两个区域提取所有未完成的任务：
   - `- [ ]` 开头的待办任务（含其所有缩进子项）
   - `- [/]` 开头的进行中任务（含其所有缩进子项）
   - 跳过 `- [x]` 已完成的任务
4. **关键规则：前一天所有未完成任务（无论来自「追踪任务」还是「今天主任务」）统一归入今天的「追踪任务」区域。「今天主任务」始终留空，由用户自己填写。**
5. **今天的日记创建在 `daily-work-item/` 根目录**（不要新建当月的 `YYYYMM/` 子目录，月份归档由用户手动完成）。检查今天的日记文件是否已存在：
   - 如果不存在：基于模板创建，任务打卡重置为全部未完成，追踪任务填入前一天所有未完成的项目
   - 如果已存在：检查是否有遗漏的未完成任务需要补充到追踪任务中
6. **打卡周期检查**：模板中「任务打卡」区标注了当前修行周期的结束日期。如果今天已超过该日期，创建日记后提醒用户：当前修行周期已结束，建议通过 /guru 与导师商定新周期并更新本模板。

created: YYYY-MM-DD

  - daily
