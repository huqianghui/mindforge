# 论文阅读：JitRL（Just-In-Time Reinforcement Learning）——把「策略改进」与「参数更新」拆开的非参数 GPI

> 接今天主任务方向一「Agent 持续学习 / test-time 自进化」（源自 [[2026-07-06-gap-analysis]] 首要行动）。JitRL 是这批 arXiv 前沿论文（JitRL / LifeSkill / XSkill / CURATOR）里最先读的一篇。
>
> 读之前带着一个问题：它自称 "Continual Learning Without Gradient Updates"，那按 [[online-learning]] 里那条硬标准（**区分 harness 与 learning，只看有没有改模型参数**），它到底是「无梯度的在线学习」还是「一个精致的 harness」？读完 + 几轮辨析后，结论比这个二选一更有意思：**JitRL 是货真价实的强化学习**——它算 advantage、做 KL 正则的策略改进，本质是教科书级的广义策略迭代（GPI）——**但它把经典 RL 焊死的两件事拆开了：「策略改进」（RL 的大脑）与「参数更新」（RL 惯用的身体）。它保留大脑、扔掉梯度身体，把学到的东西存进一个外部、持久、与权重解耦的记忆库。** 于是真正该问的不是「改没改参数」，而是「**学到的知识存在哪、什么范围生效、是否持久化**」——这根「存储位面」轴，才是理解整个持续学习簇的正确判据。

## 〇、为什么读这篇

今天的 gap-analysis 把「Agent 持续学习 / 无梯度 test-time RL」列为 arXiv 前沿盲区首位。JitRL 是切入点，因为它同时踩中三个我一直在追的问题：

1. **它和 agent-lightning 那条离线 RL 主线（系列07/08，verl + calc_x）是什么关系**？日记先验地写「互补，构成离线↔在线光谱」，读完想校准这个说法——后面第四节会发现这个"光谱"其实是二维的。
2. **它自称 "Without Gradient Updates" 的持续学习，算不算真 RL、算不算真 learning**？——这两个问题的答案不一样，第四节拆开讲。
3. **它的机制 $z'=z+\beta\hat{A}$ 看起来像给冻结模型外挂一个修正项，这和 LoRA、和 PPO/GRPO 差在哪**？

## 一、核心方法：从 (state, action, return) 到"外挂在 logits 上的 advantage"

JitRL 的持久模块不是神经网络层，而是一个**非参数经验记忆库**：

$$\mathcal{M} = \{(s_i, a_i, G_i)\}_{i=1}^{N}$$

$s_i$ 是抽象后的结构化状态，$a_i$ 是动作，$G_i$ 是该 (状态,动作) 之后观测到的累计折扣回报。整个方法是一个闭环：

**轨迹完成后（每 episode 一次）——写记忆**

1. episode 得到轨迹 $\tau = (s_1, a_1, \dots, s_T, a_T)$；
2. 一个 **LLM Evaluator** 对完整轨迹做反思式评估，映射成逐步奖励 $E: \tau \to \{r_t\}_{t=1}^{T}$；
3. 聚合成折扣回报 $G_t = \sum_{u=t}^{T} \gamma^{u-t} r_u$；
4. 把每个 transition $(s_t, a_t, G_t)$ 写入 $\mathcal{M}$。

**推理时（每次决策一次）——读记忆、改 logits**

5. 当前 observation 抽象为状态 $s$，从 $\mathcal{M}$ 取 top-$k$ 相似邻居 $N(s)$；
6. 估计状态价值与动作价值（**无 value network，纯 kNN 平均**，本质是非参数蒙特卡洛价值估计）：

$$\hat{V}(s) = \frac{1}{|N(s)|}\sum_{i \in N(s)} G_i, \qquad \hat{Q}(s,a) = \frac{1}{|N(s,a)|}\sum_{j \in N(s,a)} G_j$$

7. 得到 advantage 并归一化（数值稳定）：

$$\hat{A}(s,a) = \hat{Q}(s,a) - \hat{V}(s), \qquad \tilde{A}(s,a) = \frac{\hat{A}(s,a)}{\max_{a' \in C}|\hat{A}(s,a')| + \epsilon}$$

8. 把 advantage 作为**加性项直接加到候选动作的 logits 上**：

$$z'(s,a) = z(s,a) + \beta\,\tilde{A}(s,a)$$

reward 从不进梯度。它只通过 $G_t$ 改变**未来检索到的 $\hat{Q}$ 均值**：高 reward 的动作未来产生更高 $\hat{Q}$ → 正 advantage → logits 被上调；有害动作 → 负 advantage → 被压低。**注意一个常见误解——决策时被"重算"的不是 reward（evaluator 和折扣公式全程固定），而是随记忆累积而更新的 advantage 估计。** 这正是它便宜的原因：一次决策只是检索 + 平均，不是重新评估。

## 二、先厘清一组词：reward、return、value、advantage

理解"为什么 JitRL 是 RL 而不只是 harness"的技术命门，在于分清这四个量——尤其是 **advantage 和 reward 的区别就在一个「基线」**：

| 量 | 定义 | 性质 |
|---|---|---|
| reward $r_t$ | 环境/evaluator 给的原始标量 | **绝对**——"这一步结果多好" |
| return $G_t=\sum\gamma^{u-t}r_u$ | 从这步往后的累计折扣回报 | **绝对**——"这动作之后总共流入多少 reward" |
| value $V(s)$ | 状态 $s$ 的期望回报 | **基线**——"这个 state 平均能拿多少" |
| advantage $A(s,a)=Q(s,a)-V(s)$ | 动作相对基线的增益 | **相对**——"这动作比这个 state 的平均动作好多少" |

关键区别：**reward/return 是绝对好坏，advantage 是减掉基线后的相对好坏。** 而 advantage 这个量本身依赖一个 value 基线，**value 基线正是 RL 的核心构造（Bellman / 价值函数）**——这就是为什么在 RL 之前的 harness 世界里根本没有"advantage 估计"这回事：

- **harness（memory / RAG / reflection）只用 reward / 结果**："这条轨迹成功了，记下来"——绝对信号，不减基线。
- **RL（PPO / GRPO）用 advantage**：减基线，只推高"比局部平均更好"的动作（去掉 state 自带偏置、大幅降方差）。

**JitRL 算的是 $\hat{Q}-\hat{V}$，即 advantage——这是它在做 RL 机器、而非 harness 的铁证。** 排进"reward 怎么用"的光谱正好看清位置：**RAFT（reward 只当过滤阈值，无基线）< JitRL / PPO / GRPO（用带基线的 advantage）**。在这根轴上，JitRL 站在 PPO 那边，不在 harness 这边。

**一个具体例子：JitRL 怎么把这些量算出来。** 设回报 $G$ 是整件任务干得多好（0~1）。当前 state = "在商品列表页想找便宜货"，JitRL 检索出最像的 5 条历史经验（k=5）：

| # | 相似情境 | 当时选的 action | 回报 G |
|---|---|---|---|
| 1 | 列表页找便宜货 | 点"按价格排序" | 0.9 |
| 2 | 列表页找便宜货 | 点"按价格排序" | 0.8 |
| 3 | 列表页找便宜货 | 点第一个商品 | 0.3 |
| 4 | 列表页找便宜货 | 点"筛选" | 0.5 |
| 5 | 列表页找便宜货 | 点第一个商品 | 0.2 |

- 基线 $\hat V(s)$ = 5 条不分动作的平均 $=(0.9+0.8+0.3+0.5+0.2)/5=0.54$（"这类情境随便操作平均拿 0.54"）；
- $\hat Q(s,\text{按价格排序})=(0.9+0.8)/2=0.85$，$\hat Q(s,\text{点第一个})=(0.3+0.2)/2=0.25$，$\hat Q(s,\text{筛选})=0.50$；
- $\hat A(\text{按价格排序})=0.85-0.54=+0.31$（logits 加分），$\hat A(\text{点第一个})=0.25-0.54=-0.29$（减分）。

所以**基线不是"全状态空间的平均"，而是当前 state 的 k 个相似邻居的平均**——邻域内、混所有动作、即时求平均、零训练。

**为什么非要减这个基线——抵消"state 本身有多容易"这个公共偏置。** $\hat Q(s,a)$ 里混了两样东西：① 这个 state 整体有多好；② 这个动作在这个 state 里比别的动作强多少。① 对该 state 下所有动作都一样（所以"公共"），是个跟动作好坏无关的偏移（所以"偏置"）。减掉 ① 才剩下你真正要的 ②。用两个 state 看最清楚：

| | 简单 state（随便点都易成功） | 困难 state（怎么点都易失败） |
|---|---|---|
| 动作及 $\hat Q$ | A=0.90 / B=0.85 / C=0.80 | X=0.30 / Y=0.20 / Z=0.10 |
| $\hat V(s)$ | 0.85 | 0.20 |

直接用 raw $\hat Q$ 排序会得出"简单 state 的 C(0.80) 远好于困难 state 的 X(0.30)"——**但这是错的**：C 在自己组里最差（该避），X 在自己组里最好（该选）。raw $\hat Q$ 被"我在哪个 state"（0.85 vs 0.20 的大落差）主导，淹没了"这里哪个动作最好"。减基线后：简单组 $A_A=+0.05,A_B=0,A_C=-0.05$；困难组 $A_X=+0.10,A_Y=0,A_Z=-0.10$——X 拿到局部最高分，尽管它绝对值远低于 C。**这才对：X 是它那个处境下的最优解。**

类比**按班级平均分打分**：raw $\hat Q$ 是绝对分（普通班考 80、尖子班考 30），$\hat V(s)$ 是班级平均，advantage 是你相对本班平均高/低多少——普通班的 80 可能垫底、尖子班的 30 可能第一。判断"选得好不好"要看相对本班平均，不看绝对分。

**这对 JitRL 尤其要命**：它把 $\beta\hat A$ 加到 logits。若用 raw $\hat Q$，简单 state 的动作被整体抬高、困难 state 的被整体压低，导致困难 state 里最该选的 X 反被压得比简单 state 里最不该选的 C 还低——策略被 state 难易带偏，而非被动作好坏引导。用 advantage，加分在每个 state 内部围绕 0 居中，只反映"这动作比本地其他选项好还是差"。（这个"减公共基线"在 RL 里的正式名字叫 **variance reduction**：基线对所有动作公共，减掉它不改变改进方向的期望——无偏——却大幅降方差。）

## 三、加性更新是 KL 约束策略优化的闭式解——兼谈 KL 到底在做什么

JitRL 最硬的理论：$z' = z + \beta\hat{A}$ 不是启发式，而是下面这个目标的**闭式最优解**：

$$\pi^{*}=\arg\max_{\pi'}\Big(\underbrace{\mathbb{E}_{a\sim\pi'}[\hat{A}(s,a)]}_{\text{往高 advantage 推}}-\tfrac{1}{\beta}\underbrace{D_{KL}(\pi'\,\|\,\pi_\theta)}_{\text{别离 base 太远}}\Big)$$

**KL = Kullback–Leibler divergence（KL 散度 / 相对熵）**，衡量两个分布差多远：$D_{KL}(P\|Q)=\sum_x P(x)\log\frac{P(x)}{Q(x)}$，恒 $\ge 0$、相同才为 0、不对称。它在这里是一根**"锚 / 牵引绳"**，含两个作用：

**(a) 防止在噪声 advantage 上过度自信。** JitRL 的 advantage 来自小 memory 的 kNN，天生有噪声。没有 KL 锚，一条走运/背运的记忆就能把策略猛拽；KL 让它保守，$\beta$ 调松紧（$\beta$ 大=更信 advantage、敢偏离；$\beta$ 小=紧贴 base）。

**(b) 更深的一点：KL 正是「$z'=z+\beta\hat{A}$」这条加性更新的数学来源，也是 base 模型知识得以保留的原因。** "线性 advantage 项 − KL"这个目标的闭式解是 Gibbs/Boltzmann 形式：

$$\pi^{*}(a|s)\propto \pi_\theta(a|s)\exp(\beta\hat{A}(s,a))$$

取对数就得到 logit 空间的加性更新。**注意 $\pi_\theta$ 留在解里——是 KL 项把它保下来的。** 若没有 KL 锚，解退化成 $\pi\propto\exp(\beta\hat{A})$，base 模型先验被完全丢掉，变成"只听 memory、不听模型"。所以 **KL 项在数学上就是 JitRL「重排」候选（base logits + βA）而非「替换」候选的根本原因。**

这个 KL 和 **RLHF/PPO 里那个 KL-to-reference 惩罚是同一个东西**（防止 RL 后的模型偏离参考策略、reward hacking）——又一条把 JitRL 接回主流 RL 的线。它还内部自洽地解释了后面的局限：**KL 把 $\pi'$ 拴在 $\pi_\theta$ 的支撑集内，这既是 base 知识被保留的原因，也是 JitRL 发现不了 base 模型根本不生成的动作的原因之一。**

**和 PPO 的连接点也在这里**：这个"max advantage 期望 − KL 正则"就是 RLHF/PPO 的母目标函数。区别只在——PPO 要动参数 $\theta$，目标没有全局闭式解，只能用策略梯度 + clipping 迭代逼近；JitRL 只动单步 logits、不碰 $\theta$，目标退化成可直接求解的凸问题。**同一个目标，两种实现空间。**

## 四、它到底是不是 RL：拆开「策略改进」与「参数更新」

"它自称 learning 却不改参数"这件事之所以绕，是因为**两个不同的问题被混成一个**：

- **轴 A：在不在做 RL 式的策略改进？** → 是。算 advantage、做 KL 正则的 policy improvement，是真 RL 机器。
- **轴 B：改进有没有写回参数？** → 没有。改进存外部 memory，只在 logits 上瞬时生效。

[[online-learning]] 那条"改没改参数"的硬标准，**只在回答轴 B（结果存哪）**，它从不是在说"这里没有 RL 计算"。经典 RL 的信条是"**通过**改参数**来**改进策略"——两轴焊成一件事；**JitRL 把它们撬开了：保留 RL 的大脑（advantage-based 改进目标），扔掉 RL 惯用的身体（对权重求梯度）。**

**用 Sutton & Barto 的语言，JitRL 就是教科书级的广义策略迭代（Generalized Policy Iteration, GPI），只是搬到推理期、且非参数：**

- **策略评估**：$\hat{Q}/\hat{V}$ = memory 上 kNN 的蒙特卡洛回报平均 → 非参数 MC policy evaluation；
- **策略改进**：$z' = z + \beta\hat{A}$ = KL 正则的改进步；
- **迭代**：memory 越攒越准，策略持续改进。

所以它不是"不是 RL"，它是**以 LLM 的 logits 作为 base-policy 先验的、非参数（近乎 tabular）的强化学习**。这正好接上今天主任务方向二（Sutton & Barto 圣经）——GPI 是那本书的中轴，JitRL 是它一个不改参数的变体实现。

**顺带校准日记里"离线↔在线光谱"的说法**：JitRL 和 agent-lightning 不是一条线的两端，而是**二维平面**的两个象限——X 轴 = 改不改参数（harness ↔ learning），Y 轴 = 何时适应（离线 ↔ 部署后）。agent-lightning（verl + calc_x）在 **(改参数, 离线)**；JitRL 在 **(不改参数, 部署后)**。传统认知里"部署后 + 不改参数"这个象限只能放 memory/RAG，JitRL 证明它也能做出 RL-style 的策略改进。

## 五、三方对比：JitRL vs LoRA vs PPO/GRPO

它"看起来像 LoRA"的唯一相似点是：**都在冻结的 base model 外面叠了一个修正项**。本质完全不同。

| 维度 | JitRL | LoRA | PPO / GRPO |
|---|---|---|---|
| 外挂结构 | memory + 当前候选动作的 advantage 向量 | 低秩矩阵 $A,B$（$\Delta W = BA$） | 无外挂，直接改 $\theta$ |
| 是否低秩分解 | 否（不是矩阵） | 是 | — |
| 更新对象 | 追加 memory；推理时重排 logits | 训练 $A,B$ 参数 | 更新权重 $\theta$ |
| 更新时机 | test-time / 每次决策 | 训练或微调阶段 | 训练阶段（rollout 后 retrain） |
| 是否需要梯度 | **否** | 是（训 adapter） | 是（policy gradient） |
| 插入位置 | **logits → softmax 之前** | Transformer 权重 / attention 投影层内 | 参数本身 |
| advantage 来源 | memory 中相似轨迹的 $\hat{Q}-\hat{V}$ | — | value/critic 或 GAE（PPO）/ 组内归一化（GRPO） |
| 是否改 base model | 否，$\theta$ 冻结 | $\theta$ 冻结但引入可训练 adapter | 改 |

一句话记忆：**LoRA 改的是模型「如何思考」，JitRL 改的是当前候选动作里「谁更该被选」。**

- **LoRA 像给模型装一套可训练的外挂肌肉**——训练时用梯度练出来，之后在每层参与计算、改变 hidden representation。
- **JitRL 像给 agent 配一本「经验账本 + 即时加减分器」**——完成任务后把「当时在哪个状态、做了什么、效果如何」记进账本；下次遇到相似状态就查账本，这动作过去比平均好多少，好就给 logits 加分、差就扣分。

对 GRPO：它是 critic-less 的 PPO 变体（同一 prompt 采多个 output、用组内平均 reward 做基线算 advantage、再更新参数）。JitRL 和它共享"用 advantage 做策略改进"的语言，但 GRPO 的 advantage 进梯度、改参数，JitRL 的 advantage 进 logits、不改参数。

## 六、更好的判据：不是"改没改参数"，而是"学到的知识存在哪"

第四节说清了 JitRL 是 RL、只是把改进存在了参数之外。这就逼出一个更根本的问题——"改没改参数"这个二值判据其实是个**粗糙的投影**，真正有信息量的轴是「**学到的知识存哪、什么粒度/范围生效、是否持久化、跟权重耦不耦合**」。

先分清系统里的**两层东西**（我最初把它们混成"瞬时生效"了）：

- **logit 增量 $\beta\hat{A}$**：ephemeral——每步现算、加进 softmax、不保存，像"激活值"每次 forward 重算；
- **memory $\mathcal{M}=\{(s,a,G)\}$**：**持久的**——这才是"学到的东西"真正的载体，跨 episode 累积，存盘即跨 session。

按"存储位面"这根轴排开，各方法一目了然：

| 方法 | 学到的东西存哪 | 持久化 | 生效范围 | 与权重耦合 |
|---|---|---|---|---|
| prompt / in-context reflection | context window | ❌ 出窗口即丢 | 单会话/单轨迹 | 解耦 |
| RAG / 长期记忆 | 外部向量库 | ✅ | 全局（检索命中） | 解耦 |
| **JitRL memory** | 外部 (s,a,G) 库 | ✅ | 全局但**只在相似 state 局部**生效 | **解耦** |
| LoRA | adapter 权重 | ✅ | 全局改分布 | 绑 base 架构 |
| 全量微调 / 在线 policy gradient | base 权重 | ✅ | 永久改模型 | 绑死这个模型 |

这根轴一拉出来，JitRL 那些"看起来神奇"的性质立刻有了统一解释，而且**全都来自"知识存在外部持久库、与权重解耦"这一个事实**：

- **可移植**：换掉 base LLM，memory 照用（新模型读自己的 logits 即可）。参数级学习绑死单个模型，换模型就废。
- **可检查 / 可编辑 / 可删除**：能定点删掉某条坏经验——这正是这批论文里 **CURATOR（预算受限记忆治理 + 抗投毒）** 要解决的问题，而这个问题**只在"学习被持久化到可编辑外部库"时才存在**。微调进权重的"教训"没法外科手术式删除。
- **无灾难性遗忘**：append + retrieve，不 overwrite。

**换句话说，"存储位面"这根轴把 JitRL 从一篇孤立论文接到了整个持续学习簇上**：JitRL 定义并使用这个外部持久库，CURATOR 治理它（预算 + 抗投毒），XSkill 把它分成经验流 + 技能流两个粒度。判据从"改没改参数"换成"知识存哪"，这几篇才连成一张图——它们不是各写各的，而是在同一根轴的不同位置上做文章。

**"存储位面"轴上还有一个正交的子维度：存的东西什么粒度、要不要先编译。** 把 JitRL 和"从轨迹沉淀 skill"（XSkill 的技能流、以及 skill-library 类做法）并排，区别就显而易见——两者都是"不改参数 + 部署后 + 外部持久"，是同一象限的**兄弟**，差别只在存储的粒度与表征：

| | JitRL（经验流） | skill 沉淀（技能流） |
|---|---|---|
| 存什么 | 原始 (s,a,G) transition | 抽象出的可复用 skill / 流程 |
| 粒度 | 细（sub-symbolic 经验） | 粗（symbolic 过程） |
| 怎么用 | advantage → logits 重排 | 注入 context / 当 tool 调用 |
| 有无"压缩/抽象"步 | **无**——原样存，统计式检索 | **有**——LLM 反思轨迹→提炼成命名 skill |

一句话概括这个粒度差：**JitRL 是"惰性 / 记忆式"（像 kNN——不压缩、留全量原始经验、查询时才统计决策）；skill 沉淀是"急性 / 编译式"（把经验 compile 成可复用单元，像建索引）。** skill 路线多了一道"抽象 / 编译"工序，代价是可能丢细节、抽象错了会误导；JitRL 不编译、留原料，代价是 memory 无限膨胀（这又正好是 CURATOR 要治理的问题）。所以这三篇在存储位面轴上的分工是：**JitRL 管"存原始经验并统计使用"，XSkill 管"经验 + 技能双粒度"，CURATOR 管"外部库的预算与安全"。**

## 七、实验关键数字

| 场景 | JitRL 结果 | 对比 |
|---|---|---|
| WebArena（training-free 组） | avg 46.98% / final 51.35% | 高于 Static、Memory、Reflexion、AWM、EvoTest |
| WebArena-Lite（对比 weight-update） | final avg **60.00%** | WebRL 46.06% / SFT 23.00% |
| Jericho·Zork1 | final score **69** | EvoTest 54 / GRPO 10 |
| 成本 | ~\$290 | WebRL ~\$9,900（**省约 30×+**） |
| sample efficiency（Zork1） | 50 条轨迹达 mean 53.0 | best GRPO 需 **3,200** training episodes，mean 仅 40.7 |

一条关键 ablation：同样的 memory retrieval 下，**logit update 在 Admin/Reddit 上优于 prompt update**——说明收益不只是"把 memory 塞进 prompt"。用第六节的话说：**同一份持久记忆，作用在 logits 位面比作用在 context 位面更有效**，这是"存储/作用位面"确实是关键变量的直接证据。

## 八、局限与复现风险

- **命门是 evaluator reward 的 credit assignment**：整个 advantage 估计建立在 LLM Evaluator 把轨迹成败正确摊回每一步；credit assignment 一错，$\hat{Q}$ 就偏、logits 被带歪。
- **候选集覆盖是硬边界**：关键动作若从不进候选集（base model 采不到 + memory 里也没有），JitRL 永远发现不了它。这也是"非参数 GPI + KL 拴在 $\pi_\theta$ 支撑集"的必然结果——**它只能重排已有候选，不能创造 base 模型不生成的新动作分布**；只有真改参数才可能内化全新能力。
- **依赖状态可文本化检索**：非文本状态（图像、连续控制）不好直接套。
- **理论有假设边界**：value/advantage 一致性依赖 kNN coverage 充分、policy drift 缓慢；高噪声 reward 下不保证成立。
- **结果主要来自作者实验**：代码公开但需独立复现，上述场景下不能直接外推。

## 九、小结

- **JitRL = 冻结模型 + 外部轨迹 memory + kNN 式 advantage 估计 + logits 级闭式策略改进**。它不是新的 LoRA，也不是更便宜的 PPO，而是第三条路径。
- **它是货真价实的 RL**：用带基线的 advantage（$\hat{Q}-\hat{V}$）、做 KL 正则的策略改进，本质是**非参数的广义策略迭代（GPI）**，以 LLM logits 为 base-policy 先验。
- **它拆开了经典 RL 焊死的两件事**：策略改进（RL 的大脑，保留）与参数更新（RL 的身体，扔掉）。所以"改没改参数"这个 harness/learning 判据是个粗投影——它判的是"结果存哪"，不否认 RL 计算在发生。
- **更好的判据是"学到的知识存哪"**：存储位面 / 持久化 / 生效范围 / 与权重的耦合。JitRL 的可移植、可编辑、无灾难性遗忘，全来自"外部持久库 + 与权重解耦"。这根轴也把 JitRL、CURATOR、XSkill 连成一张图。
- **和 agent-lightning 是二维互补**：后者在 (改参数, 离线)，JitRL 在 (不改参数, 部署后)。
- **KL 项是关键**：它既是加性更新 $z'=z+\beta\hat{A}$ 的数学来源、是 base 先验被保留的原因，也是"发现不了新动作"这条局限的原因之一。
- **能力天花板 = base model 的候选覆盖**：只能重排已会的、不能创造新能力。**适用**：部署后反复做同类任务、能拿 reward、状态/动作可文本化检索；**不适用**：无反馈、状态难文本化、关键动作从不进候选集。
